Language Syntax

BASIC STRUCTURE

Every DyGram program starts with a machine declaration:

machine "My System"

// Define states
state start;
state process;
state end;

// Define connections
start -> process -> end;

TYPED CONCEPTS

Add structure and metadata to your concepts:

Concept task "User Story" {
    description<string>: "Implement feature";
    priority<Integer>: 8;
    tags: ["backend", "api"];
};

Concept implementation {
    status: "In Progress";
    owner: "Engineering";
};

task -drives-> implementation;

GENERATIVE TASKS

Integrate AI-powered tasks with prompt templates:

Input query {
    text<string>: "Analyze sentiment";
};

Task analyze {
    prompt: "Given {{ query.text }}, provide analysis";
};

Result output {
    sentiment: "TBD";
};

query -> analyze -> output;

ENTITY TYPES

DyGram supports multiple entity types:

  • Concept - Abstract ideas and domain concepts
  • Entity - Concrete domain objects
  • Task - Generative AI tasks with prompts
  • Input - System inputs and parameters
  • Result - Output values and results
  • Resource - External resources and references
  • Process - Business processes and workflows
  • Implementation - Technical implementations
  • Feature - Product features
  • Team - Teams and organizational units

TYPE ANNOTATIONS

Supported type annotations:

  • <string> - Text values
  • <Integer> - Whole numbers
  • <Float> - Decimal numbers
  • <Date> - Date values
  • <UUID> - Unique identifiers
  • <Currency> - Monetary values
  • <SemanticVersion> - Version numbers
  • <AccessLevel> - Access control levels

RELATIONSHIPS

Connect entities with labeled relationships:

// Simple connection
a -> b;

// Labeled relationship
a -drives-> b;

// Relationship with metadata
a -addresses, confidence: 0.9-> b;
