# server Command

Start a local development server with API and interactive playground.

## Usage

```bash
dy server [directory] [options]
dy serve [directory] [options]  # alias
```

## Arguments

- `[directory]` - Working directory containing machine files (default: current directory)

## Options

- `-p, --port <port>` - Port to run the server on (default: `5173`)
- `-v, --verbose` - Verbose output with request logging
- `-h, --help` - Display help

## What It Does

The `server` command starts a local development server that provides:

1. **Interactive Playground** - Web-based editor and visualizer
2. **File API** - Read/write machine files
3. **Execution API** - Execute machines via HTTP
4. **Live Reload** - Automatic updates on file changes
5. **Recording Management** - Browse and manage execution recordings

## Examples

### Basic Server

Start server in current directory:
```bash
dy server
```

Server starts at: `http://localhost:5173`

### Custom Directory

Serve machines from specific directory:
```bash
dy server ./my-machines
```

### Custom Port

Run on different port:
```bash
dy server --port 3000
```

### Verbose Mode

See request logs:
```bash
dy server --verbose
```

## Server Features

### Interactive Playground

Access at `http://localhost:5173`

Features:
- **Code Editor** - Edit DyGram files with syntax highlighting
- **Live Preview** - See visualizations in real-time
- **Execution** - Run machines directly in browser
- **File Browser** - Navigate project files
- **Error Display** - Inline error messages

### API Endpoints

The server exposes several API endpoints:

#### File Operations
- `GET /api/files/list` - List files in directory
- `GET /api/files/read?path=<path>` - Read file content
- `POST /api/files/write` - Write file content

#### Execution
- `POST /api/execute` - Execute machine
- `GET /api/recordings/list` - List recordings
- `GET /api/recordings/read?id=<id>` - Read recording

#### Checkpoints
- `GET /api/checkpoints/list` - List execution checkpoints
- `GET /api/checkpoints/read?id=<id>` - Read checkpoint

## Use Cases

### Development

Quick iteration during development:
```bash
# Terminal 1: Start server
dy server

# Terminal 2: Edit files
# Browser: See changes live at http://localhost:5173
```

### Collaboration

Share local server for team review:
```bash
dy server --port 8080
# Share: http://your-ip:8080
```

### Testing

Test machines in browser environment:
```bash
dy server ./test-machines --verbose
```

### Demos

Present machines interactively:
```bash
dy server ./demos --port 3000
```

## Development Workflow

Typical workflow with server:

```bash
# 1. Start server
dy server

# 2. Open browser to http://localhost:5173

# 3. Edit files in your editor or playground

# 4. See live updates in playground

# 5. Execute and test interactively

# 6. Save recordings for later playback
```
